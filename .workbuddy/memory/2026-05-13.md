### 🔑 重要架构改进：Pinia 用户缓存池
**问题**：PostCard/PostDetail/CommentItem 都各自从 props 或本地假数据取用户信息，同一人出现多次重复传输
**方案**：在 Pinia useUserStore 中新增 `userMap` 缓存池
- `getUser(userId)` — 按 ID 取用户信息
- `setUsers(users)` — 批量写入缓存（合并模式）
- `extractAndCacheUsers(data)` — 从帖子/评论 API 响应自动提取 user_id + username/nickname/avatar 并填充
- Home/BarDetail/PostDetail 加载列表后调用 extractAndCacheUsers 填充缓存
- PostCard/CommentItem/PostDetail 统一从 userStore.getUser() 读取
- 登录时 setUserInfo 自动将当前用户写入缓存池
- **数据流**：API 返回 → extractAndCacheUsers 入池 → 各组件 getUser() 读缓存

## 晚上：个人中心 Pinia 数据管理重构

### 问题
- Profile.vue 所有数据（帖子、评论、吧、关注/粉丝）都用本地 `ref` 管理，不共享
- N+1 请求问题：逐帖子拉评论（遍历所有帖子 fetch comments API）
- 关注/粉丝列表为空数组占位（后端有接口但前端没对接）
- EditProfile 保存后绕过 Pinia 直接操作 localStorage + reload

### 改动内容

#### 1. 后端新增 `GET /api/user/:id/comments` 接口
- 聚合查询用户所有评论，一次返回（含 post_title、用户信息），消除前端 N+1
- JOIN tb_comment + user + tb_post

#### 2. 前端 API 层新增方法（request/api/user.js）
- `getMyComments(userId)` — 走 tb-api.sawakso.com 获取用户评论
- `getFollowing(userId)` — 关注列表
- `getFollowers(userId)` — 粉丝列表

#### 3. Pinia store 大幅扩展（stores/user.js）
**新增状态**：
- `myPosts`, `myComments`, `myBars` — 个人数据列表
- `followStats` — { following, followers }
- `followingList`, `followerList` — 关注/粉丝用户列表
- `profileLoaded` — 加载标记

**新增 getters**：
- `likedPosts` — like_count > 0 的帖子（从 myPosts 派生）
- `likedComments` — like_count > 0 的评论（从 myComments 派生）
- `totalLikes` — 总获赞数

**新增 actions**：
- `fetchProfile()` — 刷新当前用户信息并 setUserInfo
- `fetchFollowStats()` — 加载关注统计
- `fetchMyPosts()` — 全量拉帖子后按 user_id 过滤
- `fetchMyComments()` — 优先走后端聚合接口，失败降级为前端 N+1
- `_fetchMyCommentsFallback()` — 降级方案
- `fetchMyBars()` / `fetchFollowing()` / `fetchFollowers()`
- `unfollowUser(targetUser)` — 取关+更新本地状态
- `fetchAllProfileData()` — 一键加载全部个人中心数据（并行）

**logout 增强**：登出时清空所有个人中心数据

#### 4. Profile.vue 全面精简
- **移除**：profile, myPosts, myComments, myBars, likedPosts, likedComments, allPosts, followStats, followingList, followerList 等全部本地 ref
- **移除**：fetchProfile, fetchFollowStats, totalLikes computed, getPostTitle, handleUnfollowUser 等函数
- **改为**：统一从 userStore 读取（userStore.myPosts, userStore.followStats.following 等）
- **onMounted**：只需一行 `await userStore.fetchAllProfileData()`
- **取关**：调用 `userStore.unfollowUser(u)`
- **编辑资料回调**：调用 `userStore.fetchProfile()` 刷新
